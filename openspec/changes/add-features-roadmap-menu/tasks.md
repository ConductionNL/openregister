- [x] 1.3 Create `lib/Controller/GitHubIssuesController.php` with:
  - `index()` action mapped to `GET /api/github/issues` — proxies to `listIssues()`.
  - `create()` action mapped to `POST /api/github/issues` — validates title (3-200 chars), body (>=10 chars), repo regex `^[\w.-]+/[\w.-]+$`, applies rate limit, delegates to `createIssue()`.
- [x] 1.4 Register both routes in `appinfo/routes.php`. Apply attribute declarations per ADR-005: GET carries `#[NoAdminRequired]` + `#[NoCSRFRequired]`; POST carries `#[NoAdminRequired]` but NOT `#[NoCSRFRequired]` (CSRF MUST apply).
- [x] 1.5 Implement in-memory TTL cache (15 min) keyed by `(repo, state, sort, per_page)` for GET only; set `X-OpenRegister-GitHub-Cache: HIT|MISS` response header.
- [x] 1.6 Implement APCu-backed per-user rate limit for POST — key `openregister.feature_submission:<user_id>`, 60s TTL; return HTTP 429 with `{error: "rate_limited", retry_after: <seconds>}` when exceeded.
- [x] 1.7 Graceful degradation: GET returns HTTP 200 `{items: [], hint: "github_pat_not_configured"}` when no app-level PAT set; POST returns HTTP 503 `{error: "github_pat_not_configured"}` when neither user nor app-level PAT is configured.
- [x] 1.8 specRef handling in `create()`: when provided, append `\n\n---\nRelated capability: `<specRef>`\n` to body AND apply `specRef:<slug>` label on the created issue.
- [x] 1.9 Map GitHub rate-limit (403 with `X-RateLimit-Remaining: 0`, or 429) to HTTP 429 with `{error: "github_rate_limited", reset_at: "<ISO-8601>"}` derived from `X-RateLimit-Reset`.
- [x] 1.10 Strip sensitive fields from issue payloads before returning (keep: `number`, `title`, `body`, `html_url`, `user.login`, `user.avatar_url`, `reactions.total_count`, `reactions.+1`, `created_at`, `updated_at`, `labels[].{name,color}`); drop everything else.
- [x] 1.11 Add PHPUnit tests for the controller and both handler methods covering: GET success, GET graceful-degraded, GET cache HIT, GET rate-limit mapping, GET invalid repo/per_page rejection, POST success with user PAT (no prefix), POST success with server PAT (with prefix), POST with specRef (body suffix + label), POST rate limit 429, POST missing-both-PATs 503, POST validation errors (title/body/repo), and PAT-leak assertion — the placeholder `YOUR_API_KEY_HERE` must not appear in any response body, header, log, or cache entry.

- [ ] 1.13 Run `composer check:strict` (PHPCS, PHPMD, Psalm, PHPStan) and fix any findings in the new files AND any pre-existing issues encountered.
- [x] 1.14 **Repo allowlist enforcement (security):** read `openregister::github_repo` from IAppConfig at the start of both `index()` and `create()`; reject any request whose `repo` parameter/body field does not equal the configured value with HTTP 403 `repo_not_allowed`; when the key is unset, return HTTP 200 `{items: [], hint: "github_repo_not_configured"}` (GET) or HTTP 503 `{error: "github_repo_not_configured"}` (POST). Add PHPUnit cases: `repo_not_allowed` 403 on cross-repo POST, `repo_not_allowed` 403 on cross-repo GET, `github_repo_not_configured` graceful response on unset config.
- [x] 1.15 **Display-name + instance-URL sanitization (security):** in `GitHubHandler::createIssue`'s server-PAT-fallback branch, sanitize the display name (strip `\r`, `\n`, `*`, `_`, `[`, `]`, `(`, `)`, `` ` ``, `<`, `>`, `\` — replace with spaces — then truncate to 80 chars) and validate the instance URL begins with `https://` (or `http://localhost`); on scheme failure, fall back to the literal `> Submitted via Nextcloud OpenRegister` block without embedding the URL. Add PHPUnit cases: markdown-injection display name produces sanitized output, separator `\n\n---\n\n` appears exactly once, oversized display name is truncated to 80 chars, non-https instance URL falls back to generic prefix.
- [x] 1.16 **specRef format validation (security):** before delegating to `createIssue`, validate `specRef` matches `^[a-z0-9][a-z0-9-]*[a-z0-9]$` with length ≤ 80 when present; reject with HTTP 400 `specref_invalid_format`. Add PHPUnit cases: invalid characters, embedded newline, length > 80, valid kebab-case slug accepted, empty/absent value accepted.
- [x] 1.17 **Audit logging for server-PAT submissions (security):** when `createIssue` falls through to the server PAT path AND the GitHub call succeeds (201), emit ONE INFO entry to `app.log` with exactly the fields `{user_id, repo, issue_number, specref, timestamp}`; on failure, emit a WARNING entry with the same fields plus `{error_code, github_status}`. Never log PAT, body, or title. Add PHPUnit cases: success path emits exactly one INFO entry with no PAT/body/title in the message, failure path emits exactly one WARNING with the error code, user-PAT path emits NO `openregister`-app entry.
- [x] 1.18 **APCu-unavailable rate-limit fallback (resilience):** abstract the rate-limiter behind an interface; primary impl uses APCu, fallback impl uses Nextcloud's `ICacheFactory::createDistributed()` / `createLocking()`; when neither is available, fail closed with HTTP 503 `rate_limiter_unavailable`. Apply to BOTH the POST submission limiter AND the new GET cache-miss limiter. Add PHPUnit cases: APCu disabled + distributed cache available → 429 still fires on second submission within 60s; both unavailable → 503 `rate_limiter_unavailable`.
- [x] 1.19 **Per-user GET rate limit (resilience):** instrument the GET endpoint with a per-user cache-miss counter (max 10 distinct cache keys per user per 5 min); cache hits do not count; on exceed, respond HTTP 429 `{error: "user_rate_limited", retry_after}`. Add PHPUnit case: 11th distinct cache-miss request from the same user in 5 min returns 429.
- [x] 1.20 **`sort` parameter allowlist (resilience):** constrain `sort` to `{"reactions-+1", "created", "updated", "comments"}`; reject other values with HTTP 400 `sort_invalid_value`. Add PHPUnit case: `sort=stars` returns 400 `sort_invalid_value`.
- [x] 1.20b **`labels` parameter validation (security + resilience):** in `index()`, parse `labels` as a comma-separated list, validate each entry matches `^[a-z][a-z0-9_-]*$` with length ≤ 50, reject with HTTP 400 `labels_invalid_format` on mismatch. Reject lists with more than 8 entries with HTTP 400 `labels_too_many`. Add PHPUnit cases: SQL-injection-shaped label (`foo;DROP TABLE`) → 400, label exceeding 50 chars → 400, 9-entry list → 400, valid 2-entry list (`enhancement,feature`) accepted and results merged via two GitHub calls deduped by issue number. Update the cache key to include the (sorted) labels list so different filters do not collide on the same cache slot.
- [x] 1.21 **Admin opt-out (compliance):** read `openregister::features_roadmap_enabled` (default `true`) at the start of both `index()` and `create()`; when `false`, return HTTP 403 `{error: "feature_disabled"}` and skip all downstream logic. Add PHPUnit case: flag false → both endpoints return 403.
- [x] 1.22 **Admin documentation (PAT scopes + token lifecycle):** in OpenRegister's admin docs section for the GitHub integration, document the minimum required PAT scope (`public_repo`), the recommendation to prefer fine-grained PATs with explicit expiry, the recommended rotation cadence (90 days), and the revocation procedure on suspected compromise. Add a section explaining the `openregister::github_repo` allowlist key and the `openregister::features_roadmap_enabled` opt-out flag.
